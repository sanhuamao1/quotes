# 产品说明书：词摘 - 读书笔记摘抄应用

## 一、产品概述

### 1.1 产品定位

**词摘**是一款专为文字创作者设计的移动端读书笔记工具，帮助用户在阅读过程中快速摘抄优美的词语、句子和段落，建立个人词汇库，为小说创作积累素材。

### 1.2 核心目标用户

- 小说创作者
- 文学爱好者
- 需要积累写作素材的阅读者

### 1.3 核心价值主张

- **快速记录**：随时随地记录触动心灵的文字
- **灵活分类**：通过标签自由组织摘抄内容
- **高效检索**：多标签组合检索，快速找到所需素材

---

## 二、功能需求

### 2.1 核心功能模块

#### 2.1.1 摘抄记录

| 功能项   | 说明                           |
| -------- | ------------------------------ |
| 内容输入 | 纯文本输入，支持多行文本       |
| 标签添加 | 自由输入标签，支持添加多个标签 |
| 快捷操作 | 保存后自动清空，可快速记录多条 |
| 时间戳   | 自动记录创建时间               |

#### 2.1.2 标签系统

| 功能项     | 说明                       |
| ---------- | -------------------------- |
| 自由标签   | 用户自定义输入任意标签     |
| 标签联想   | 输入时显示历史使用过的标签 |
| 多标签支持 | 单条摘抄可添加多个标签     |
| 标签管理   | 可查看所有标签及其使用次数 |

#### 2.1.3 检索浏览

| 功能项     | 说明                       |
| ---------- | -------------------------- |
| 多标签筛选 | 可选择多个标签进行组合检索 |
| 全部摘抄   | 按时间倒序浏览所有记录     |
| 随机浏览   | 随机展示摘抄，激发灵感     |
| 全文搜索   | 支持关键词搜索摘抄内容     |

### 2.2 页面结构

```
┌─────────────────────────────────────────┐
│  底部导航栏：记录 | 浏览 | 标签 | 我的    │
└─────────────────────────────────────────┘
```

#### 页面1：记录页（首页）

- 大文本输入框：输入摘抄内容
- 标签输入区：添加/选择标签
- 保存按钮

#### 页面2：浏览页

- 顶部：搜索框 + 标签筛选区
- 主体：摘抄列表卡片
- 每张卡片：内容 + 标签 + 时间

#### 页面3：标签页

- 所有标签云/列表
- 点击标签进入该标签下的摘抄列表

#### 页面4：我的

- 数据导出功能
- 设置

## 三、交互设计

### 3.1 记录流程

```
打开APP → 输入摘抄内容 → 添加标签 → 保存 → 清空准备下一条
```

### 3.2 检索流程

```
进入浏览页 → 点击标签筛选 → 选择多个标签 → 查看结果
         → 或使用搜索框输入关键词
```

### 3.3 标签交互

- **添加标签**：输入文字后按空格确认
- **删除标签**：点击标签上的×
- **标签联想**：输入前2个字符时显示匹配的历史标签

## 五、UI/UX 设计要点

### 5.1 视觉风格

- **风格**：简洁、文艺、沉浸式阅读体验
- **配色**：浅色纸张质感背景，深色文字
- **字体**：正文使用宋体/仿宋类字体，增强文学感

### 5.2 关键设计细节

1. **记录页**：输入框占据主要区域，减少干扰
2. **标签展示**：使用胶囊样式，颜色可自定义或按标签分配
3. **摘抄卡片**：留白充足，突出文字内容本身
4. **空状态**：鼓励性文案引导用户开始记录

### 5.3 响应式考虑

- 适配各种手机屏幕尺寸
- 横屏时自动调整为双栏布局（可选）

## 七、技术建议

### 7.1 技术选型

- **框架**：TaroJS + React（支持微信小程序/H5/APP多端）
- **UI组件库**：Taro UI 或 自定义组件
- **状态管理**：React Hooks 或 Zustand

### 7.2 性能考虑

- 摘抄列表使用虚拟滚动（记录超过1000条时）
- 标签检索建立索引

## 八、使用场景示例

### 场景1：日常积累

> 读者小王在看《红楼梦》，看到"寒塘渡鹤影，冷月葬花魂"觉得意境很美，打开"词摘"APP，输入这句话，添加标签"#意境"、"#诗句"、"#悲情"，保存。

### 场景2：写作时检索

> 小王正在写小说，需要描写一个紧张的对话场景。他打开APP，在浏览页选择标签"#对话"、"#紧张"，系统筛选出所有同时带有这两个标签的摘抄，从中获得灵感。

### 场景3：标签整理

> 周末整理笔记，小王在标签页看到之前用的"好看"这个标签太笼统，决定把内容重新标记为"#修辞"、"#比喻"等更精确的标签。
